## `doc/` Folder – Extended Documentation

This folder is reserved for **deeper documentation** that goes beyond the core project files in `project/`.

Current project‑level docs:

- `agent.md` – AI constraint dossier.
- `readme.md` – Operational guide and grading checklist.
- `project.md` – Product scope and user stories.
- `module.md` – Technical architecture and module breakdown.

Use `doc/` for:

- Additional architecture diagrams and design notes.
- UI wireframes or screenshots.
- Backend and security design documents for future phases.

Planned (optional) files:

- `architecture.md` – Detailed diagrams and flow descriptions for the full system once a backend is added.
- `ui-wireframes.md` – Sketches or images of key screens.
- `future-backend.md` – Expanded Zero‑Trust backend design, database schema details, and deployment notes.

For the MVP, these documents are **not required for grading**. The folder simply signals where extended docs should live as the project grows.
